Inside the run function, implement these sequential stages:

**Stage 1 â€” Fetch data from Supabase (per D-20, D-28):**
- `metadata.set('step', 'Buscando dados para sintese...')` and `metadata.set('progress', 0)`
- Fetch in parallel: `const [analysis, competitors, viralContent] = await Promise.all([getAnalysis(payload.analysisId), getCompetitorsByAnalysis(payload.analysisId), getViralContentByAnalysis(payload.analysisId)])`
- Extract `viralPatterns` from `analysis?.viralPatterns ?? null`
- If `!analysis`: return `{ status: 'unavailable', data: { synthesis: null, scripts: null }, reason: 'Analise nao encontrada no banco de dados' }`

**Stage 2 â€” Call synthesizeAnalysis (per D-15, D-17):**
- `metadata.set('step', 'Gerando sintese e recomendacoes...')` and `metadata.set('progress', 30)`
- Call `synthesizeAnalysis({ niche: payload.niche, segment: payload.segment, region: payload.region, competitors, viralContent, viralPatterns })`
- Store result as `synthesisResult`

**Stage 3 â€” Call generateCreativeScripts (per D-15, D-17 â€” after synthesis):**
- `metadata.set('step', 'Gerando roteiros criativos...')` and `metadata.set('progress', 60)`
- Call `generateCreativeScripts({ niche: payload.niche, segment: payload.segment, viralContent, viralPatterns, competitors })`
- Store result as `scriptsResult`

**Stage 4 â€” Determine status and store (per D-23, D-24, D-31):**
- `metadata.set('step', 'Salvando resultados...')` and `metadata.set('progress', 85)`
- Determine status:
  - Both non-null: `'success'`
  - One null: `'partial'`
  - Both null: try fallback (per D-22) â€” build a minimal fallback synthesis from raw competitor data as stringified summary, set status to `'fallback'`. If even that fails, `'unavailable'`
- Build the `strategicOverview` string for DB: `JSON.stringify(synthesisResult)` (sections object as JSON string in TEXT column) if synthesis succeeded, or a fallback summary string if not
- Call `upsertSynthesis({ analysisId: payload.analysisId, strategicOverview, recommendations: synthesisResult?.recommendations ?? [], creativeScripts: scriptsResult ?? [] })`
- `metadata.set('progress', 100)`
- Return `{ status, data: { synthesis: synthesisResult, scripts: scriptsResult }, reason }` where `reason` is only set for non-success statuses

The fallback synthesis (per D-22) when Gemini completely unavailable:
- Build a simple string: `"Sintese automatica indisponivel. Dados coletados: ${competitors.length} concorrentes analisados, ${viralContent.length} conteudos virais identificados. Consulte os dados individuais no dashboard."`
- Store empty recommendations array and empty scripts array
- This ensures the user at least sees extraction data even without AI synthesis (per D-27)
