The task structure follows 6 sequential stages (per D-36) with parallel operations within each stage:

```typescript
import { task, metadata } from '@trigger.dev/sdk';

import { searchViralTiktok } from '@/lib/apify/tiktok-viral';
import { searchViralInstagram } from '@/lib/apify/instagram-viral';
import { calculateEngagementRate } from '@/lib/apify/tiktok-viral';
import { uploadFile } from '@/lib/storage/bunny';
import { transcribeVideo } from '@/lib/transcription/transcribe';
import { extractHookBodyCta } from '@/lib/ai/hbc-extraction';
import { detectViralPatterns, type PatternDetectionInput } from '@/lib/ai/viral-patterns';
import {
  createViralContent,
  updateViralContent,
  updateAnalysisViralPatterns,
} from '@/lib/supabase/queries';
import type {
  ViralContent,
  ViralVideoCandidate,
  ViralPatterns,
  HookBodyCta,
} from '@/types/viral';

/** Payload para extracao de conteudo viral do nicho */
export interface ExtractViralPayload {
  analysisId: string;
  niche: string;
  segment: string;
  region: string;
}

/** Resultado do pipeline de extracao viral */
export interface ExtractViralResult {
  status: 'success' | 'partial' | 'unavailable';
  data: {
    viralContent: ViralContent[];
    patterns: ViralPatterns | null;
  };
  reason?: string;
}

/** Estado de progresso do pipeline viral (per D-50) */
interface ViralProgress {
  discover: 'pending' | 'running' | 'completed' | 'failed';
  filter: 'pending' | 'running' | 'completed';
  download: 'pending' | 'running' | string | 'completed' | 'failed';
  transcribe: 'pending' | 'running' | string | 'completed' | 'failed';
  hbc: 'pending' | 'running' | string | 'completed' | 'failed';
  patterns: 'pending' | 'running' | 'completed' | 'skipped' | 'failed';
}
```

The `extractViral` task config:
- `id: 'extract-viral'`
- `maxDuration: 300` (per research recommendation, increased from stub's 180)
- `retry: { maxAttempts: 2, minTimeoutInMs: 3000, maxTimeoutInMs: 8000, factor: 2 }` (per D-39)
- Returns `ExtractViralResult`

**Stage 1: Discover** (per D-36 step 1)
- Set progress: `{ discover: 'running', ... all others: 'pending' }`
- Run `searchViralTiktok(niche, segment)` and `searchViralInstagram(niche, segment)` via `Promise.allSettled` (per D-40, D-41)
- Extract results: fulfilled = candidates[], rejected = empty[] + warning
- If BOTH fail: set progress discover='failed', return `{ status: 'unavailable', data: { viralContent: [], patterns: null }, reason: 'Nenhuma plataforma retornou resultados para o nicho' }`
- Merge TikTok + Instagram candidates into single array
- Set progress: discover='completed'

**Stage 2: Filter** (per D-36 step 2)
- Set progress: filter='running'
- Already filtered by wrappers (duration <= 240s, date, engagement sort, top 5 per platform), but merge here creates combined pool
- If 0 candidates after merge: return unavailable
- Set progress: filter='completed'

**Stage 3: Download** (per D-36 step 3)
- Set progress: download='running'
- For each candidate in candidates array (use `for...of` loop to avoid holding all buffers in memory simultaneously, per research recommendation):
  1. `fetch(candidate.videoUrl)` to get video as Buffer
  2. If fetch fails: log warning, skip to next candidate (per D-18)
  3. Upload to Bunny: `uploadFile(`viral/${analysisId}/${candidate.platform}/${String(index + 1).padStart(2, '0')}.mp4`, buffer)` (per D-16)
  4. Create DB record immediately via `createViralContent({...candidate fields, bunnyUrl: cdnUrl})` (per D-19)
  5. Update progress: `download='${successCount}/${totalCount}'`
- After loop: if 0 downloads succeeded, return `{ status: 'unavailable', data: { viralContent: [], patterns: null }, reason: 'Nenhum video pode ser baixado' }`
- Set progress: download='completed'
- Keep array of successfully downloaded `ViralContent` records

**Stage 4: Transcribe** (per D-36 step 4)
- Set progress: transcribe='running'
- Use `Promise.allSettled` to transcribe all downloaded videos in parallel (per D-22)
- For each: call `transcribeVideo(record.bunnyUrl!)` using Bunny CDN URL (per D-21)
- For fulfilled results: `updateViralContent(record.id, { transcription: result.text })` (per D-23)
- For rejected results: log warning, video keeps null transcription (per D-24)
- Update progress: `transcribe='${successCount}/${totalCount}'`
- Set progress: transcribe='completed'
- Note: AssemblyAI free tier allows 5 streams/min. If more than 5 videos, batch into groups of 5 with 2s delay between groups.

**Stage 5: Per-video HBC** (per D-36 step 5)
- Set progress: hbc='running'
- Filter to only videos WITH transcription (per D-29)
- Use `Promise.allSettled` to run `extractHookBodyCta(transcription, durationSeconds)` in parallel (per D-28)
- For fulfilled non-null results: `updateViralContent(record.id, { hookBodyCta: result })` (per D-27)
- For rejected/null results: log warning, video keeps null hookBodyCta (per D-44)
- Update progress: `hbc='${successCount}/${totalCount}'`
- Set progress: hbc='completed'

**Stage 6: Cross-video Patterns** (per D-36 step 6)
- Count transcribed videos (those with non-null transcription)
- If fewer than 2: set progress patterns='skipped', patterns=null (per D-48)
- Otherwise: set progress patterns='running'
- Build `PatternDetectionInput[]` from transcribed videos: `{ transcription, platform, caption, durationSeconds, engagementRate }`
- Call `detectViralPatterns(inputs)` (per D-30 — single batch call)
- If result: `updateAnalysisViralPatterns(analysisId, result)` (per D-33)
- Set progress: patterns='completed'

**Return** (per D-38):
- Determine status: all stages succeeded + all videos transcribed = 'success'; at least 1 video with data = 'partial'; nothing = 'unavailable'
- Return `{ status, data: { viralContent: downloadedRecords, patterns }, reason? }`

**Error handling:**
- Wrap entire run in try/catch (per error handling strategy golden rule D-45)
- On unhandled error: set metadata status='failed', return `{ status: 'unavailable', data: { viralContent: [], patterns: null }, reason: errorMessage }`
- All error messages in PT-BR (per D-51)

**Helper functions to extract from the main run (keep under 30 lines per function per CLAUDE.md):**
- `downloadAndStoreVideo(candidate, analysisId, index)`: fetch -> buffer -> uploadFile -> createViralContent -> return ViralContent | null
- `transcribeVideoSafe(record)`: try transcribeVideo, catch -> null. Returns `{ recordId, transcription, durationSeconds } | null`
- `extractHbcSafe(record, transcription)`: try extractHookBodyCta, catch -> null. Returns `HookBodyCta | null`
- `determineStatus(downloaded, transcribed, hbcExtracted)`: logic to determine success/partial/unavailable
